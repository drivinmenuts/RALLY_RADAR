# Rally Radar

Daily metal-detecting rally scraper for **PE15 9HD** (March, Cambridgeshire).

Scrapes 7 public sites plus 3 Facebook pages, geocodes each event, filters to a 14-day / 200-mile window, writes `events.json`, and sends a digest by email and/or WhatsApp.

The HTML dashboard (`detecting-rally-radar.html`) reads `events.json` from this repo's `main` branch raw URL and renders it with a radius slider, sortable table, Leaflet map, and ICS/CSV/JSON export.

## One-time setup

### 1. Push everything to GitHub

```bash
git init
git remote add origin https://github.com/Drivinmenuts/RALLY_RADAR.git
git add .
git commit -m "Initial Rally Radar"
git push -u origin main
```

### 2. Add secrets

In your GitHub repo: **Settings → Secrets and variables → Actions → New repository secret**

Add each of these:

| Secret name        | Value                                                              |
|--------------------|--------------------------------------------------------------------|
| `WEB3FORMS_KEY`    | `755f72cf-f65b-4e0d-ba8d-b92561bdaed9`                             |
| `EMAIL_TO`         | the email address that should receive the digest                   |
| `CALLMEBOT_PHONE`  | your mobile in international format, e.g. `447123456789`           |
| `CALLMEBOT_KEY`    | the API key CallMeBot gives you (see step 3)                       |
| `FB_COOKIE`        | *(optional)* Facebook session cookies as JSON, if FB starts gating |

### 3. Activate CallMeBot WhatsApp

1. Add **+34 644 51 95 23** to your phone contacts as "CallMeBot".
2. Send "I allow callmebot to send me messages" to that number via WhatsApp.
3. Wait for the reply — it contains your personal API key.
4. Paste the key into the `CALLMEBOT_KEY` secret above.

(Full instructions at https://www.callmebot.com/blog/free-api-whatsapp-messages/)

### 4. Run it once manually

Go to the **Actions** tab in GitHub → **Rally Radar - Daily Sweep** → **Run workflow**.

After ~5 minutes it will commit a fresh `events.json` and send you a test digest.

### 5. View the dashboard

Open `detecting-rally-radar.html` in any browser. It pulls the live `events.json` from your repo automatically.

For a permanent URL, drop the HTML on GitHub Pages or any static host.

## Schedule

The workflow runs at **07:00 UTC daily** (`.github/workflows/rally-radar.yml`, line `cron: '0 7 * * *'`).

Edit that line to change the schedule. Cron format: `minute hour day-of-month month day-of-week`.

## How geocoding works

1. UK postcodes → **postcodes.io** (free, no key)
2. Place names → **Nominatim** (OSM, 1 req/sec, polite UA)
3. Results cached in `scraper/cache.json` so we don't repeat lookups

## Maintenance

**FB scraper breaks** (most likely failure):
Facebook changes their DOM ~quarterly. If the FB sources start returning zero, check the Action logs and update the selectors in `scraper/sources.mjs` (`scrapeFacebookPages`).

**A web source restructures**:
Same story — update the relevant `scrape*()` function in `sources.mjs`. The parsers are intentionally permissive so most reshuffles still work.

**Wrong location**:
Edit `scraper/cache.json` directly to override a stale geocode, or add a manual override block to `scraper.mjs` (TODO).
